### 2. 赛道选择的战略考量

我们共同分析了"挑战组"和"初创组"的利弊。考虑到以"求职"为最终导向，一个能够直接体现硬核技术能力的实战项目，远比一份商业计划书更有说服力。因此，我们果断地选择了更具技术挑战性的**【挑战组 · 算法赛】**。

在挑战组的四个赛题中，我们进行了进一步的筛选：

*   **排除项**: 赛题三（长思维链推理路径优化）因其偏向底层算法，对学历和研究背景要求较高，与我们的求职目标和背景不太匹配。
*   **最终选择**: **【赛题四：多模态财报自动生成】**

这个选择是基于以下几点深思熟虑的判断：

1.  **高度对齐**: 该赛题是一个典型的、完整的 AI Agent 应用，完美契合我们"Agent/RAG"的目标方向。
2.  **扬长避短**: 赛题偏向应用层构建，更能发挥您对"自动化"的理解和热情，以及产品构建的潜力，而不是去硬碰底层算法。
3.  **求职导向**: 这个项目经验对于应聘本科即可的 AI 应用开发岗，具有极高的价值和吸引力。

## 三、作战计划：解构"多模态财报生成 Agent"

方向既定，我们便将这个宏大的目标拆解成了可执行的模块化任务，为后续的技术攻坚绘制了清晰的"作战地图"：

1.  **感知模块 (Perception)**: Agent 的"眼睛"。负责从财报（PDF）中精准提取**文字、表格、图表**三类信息。
    *   *关键技术*: OCR, 表格/图表识别, 文档解析。

2.  **认知模块 (Cognition/Reasoning)**: Agent 的"大脑"。负责理解、关联、推理提取出的信息，进行财务分析。
    *   *关键技术*: Prompt Engineering, CoT, ReAct, RAG。

3.  **执行/生成模块 (Action/Generation)**: Agent 的"手"和"嘴"。负责将分析结果生成结构化的专业研报。
    *   *关键技术*: 长文本生成, NLG。

4.  **调度/编排模块 (Orchestration)**: Agent 的"神经中枢"。负责编排整个工作流，确保各模块高效协同。
    *   *关键技术*: Agent 框架 (如 LangChain, LlamaIndex)。
